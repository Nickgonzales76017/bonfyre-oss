Binary reference · Layer 5

Revenue & ops.

Everything between producing work and getting paid for it, tracked. Eight binaries: dynamic pricing pages, Stripe-compatible checkout, per-usage metering, append-only financial ledger, margin reports, follow-up sequencing, project tracking, and quantitative modelling. None phone home. None have a SaaS markup. None require a cloud service to run.

What this layer eliminates

The stack you're currently paying to replace this with.

Stripe Billing + usage metering

$0.8% of revenue above Stripe's cut

Stripe Billing adds 0.5–0.8% on top of standard processing fees for usage-based billing. BonfyreMeter records usage events in LMDB at zero per-event cost. You still pay Stripe's base processing fee — you just don't pay a SaaS margin on top of it.

Outreach / Apollo sequences

$100–$150/user/month

BonfyreOutreach sequences follow-up messages from pipeline outputs. No per-seat license. Sequence logic is config — YAML or JSON rules evaluated at pipeline time, not in a cloud CRM.

QuickBooks / Xero financial reporting

$30–$80/month subscription

BonfyreLedger maintains an append-only financial log. BonfyreFinance reads it and produces margin reports in JSON or PDF via BonfyreEmit. No subscription. The ledger is a plain file — importable into any accounting tool if you ever need it.

Binary reference

All 8 Layer 5 binaries.

BonfyreOffer

Dynamic pricing surface generation

Takes a brief JSON and a pricing rules file, and generates a structured offer: service tiers, pricing amounts (computed from brief context like project scope, materials mentioned, hours implied), payment terms, and an expiry date. Output is a proposal object — machine-readable, not just a document — that BonfyrePay can attach a checkout session to directly. Pricing rules are YAML — you define margins, discounts, and tier thresholds without modifying code.

BonfyrePay

Checkout and payment handling

Generates Stripe-compatible checkout sessions from BonfyreOffer proposals. Handles: one-time payments, subscription setup, and usage-based billing activation. On session completion, writes a payment confirmation to BonfyreLedger. On failure, writes a failure event. Does not store card data — passes through to Stripe's PCI-compliant handling. The checkout URL returned is a standard Stripe Checkout hosted page.

BonfyreMeter

Usage metering and quota tracking

Records per-usage events against named meters in LMDB: API call counts, processing minutes, storage bytes, seat counts, or any custom event type. Meters have configurable quotas and billing periods. When a quota is reached, BonfyreMeter emits a quota-exceeded event that BonfyreGate (Layer 6) can act on to rate-limit or block the relevant account. Meter data is the source for BonfyreFinance usage reports.

BonfyreLedger

Immutable append-only financial audit trail

Append-only log of all financial events: payments received, refunds issued, subscription activations, quota overages, and manual adjustments. Each entry is timestamped and linked to a source artifact ID (the BonfyrePack bundle that generated the billable event). The log is content-addressed and signed — entries cannot be backdated or silently modified. BonfyreFinance reads this ledger; it never writes to it.

BonfyreFinance

Margin calculation and financial reporting

Reads BonfyreLedger and BonfyreMeter data to produce structured financial reports: gross revenue by period, margin per offer type, usage cost breakdown, and projected run rate. Output formats are JSON (for programmatic use), a formatted PDF report via BonfyreEmit, and a CSV export for import into external accounting tools. Runs on-demand — point it at the ledger file and it computes immediately, no running service required.

BonfyreQuant

Quantitative analysis and projection modelling

Statistical analysis and scenario modelling over financial and usage time series. Forecasts revenue based on configurable growth assumptions, models quota usage against projected capacity, and performs sensitivity analysis (how does 20% churn affect the 6-month projection?). Uses LAPACK-compatible linear algebra over the ledger data. Output is a JSON report with named scenarios and confidence intervals.

BonfyreOutreach

Follow-up sequencing and delivery

Schedules and delivers follow-up messages from pipeline outputs. Sequence rules are defined in YAML: "3 days after offer sent with no response: send reminder. 7 days: send expiry warning. 14 days: close." Messages are templated from the offer or brief context. Delivery is via BonfyreTel (email/webhook). Tracking records whether each message was sent and whether a response event was received. No per-seat license. Sequence logic lives in config files, not a cloud platform.

BonfyreProject

Action item and project state tracking

Creates and tracks project items sourced from BonfyreBrief action item fields: owner, deadline, status (open/in-progress/resolved/blocked), source artifact link, and notes. Items are stored in LMDB, queryable via BonfyreQuery, and renderable via BonfyreRender as a project board page. When BonfyreTel confirms delivery of a committed action, BonfyreProject marks the item resolved. The project board is a static HTML page — no project management tool subscription required.

What comes next

Layer 6 makes everything composable at scale.

Output of Layer 5

Payments tracked, usage metered, follow-ups sequenced

A signed ledger of payments, a metered usage database, scheduled follow-up sequences, and tracked action items. Everything needed to run revenue operations without a cloud billing platform.

Input to Layer 6

BonfyreGate controls access based on meter state

Layer 6's BonfyreGate reads Layer 5's BonfyreMeter quotas. When a quota is exceeded, Gate blocks further pipeline processing or API requests for the relevant account — without a separate policy service.

Full pipeline

Layer 1 through 5 in one run

A single BonfyrePipeline definition can chain all five layers: ingest a recording, transcribe, brief, embed, offer, pay, meter, and deliver — triggered by one file drop. No manual intervention between steps.
