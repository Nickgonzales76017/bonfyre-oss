bonfyre-run

Every pipeline here has a shortcode. One command runs it.

bonfyre-recipe stores a SQLite registry of pipeline recipes. bonfyre-run resolves any code, validates the binaries on your PATH, executes the stage DAG — parallel where safe — and writes a run-manifest.json receipt when it's done. Type it once, get the whole pipeline.

What happens when you run it

Plan → validate → execute → manifest

Every run follows the same four steps. First, bonfyre-run resolves the recipe from the SQLite registry (or from built-ins). Then it validates that each stage binary actually exists on your PATH — so you find out about missing binaries before the pipeline starts, not halfway through. Then it assigns topological levels and forks all stages at the same level concurrently. When done, a run-manifest.json lands in the output directory with each stage's exit code, wall time, and overall status.

Use --dry-run to see the full execution plan before committing. Use --from-stage and --to-stage to re-run a slice of the pipeline. Use --batch DIR to process a whole directory of recordings with one command.
