Beyond pipelines: a system that improves itself.

The original 53 Bonfyre binaries handle ingestion, transcription, tagging, summarization, distribution, and delivery. They are excellent at doing exactly what they're told. What they don't do — by design — is decide how to do it, whether the result is good enough, or what to try next when it isn't.

The control layer is that missing brain. It evaluates every artifact on the six HE-SLI dimensions (relevance, completeness, coherence, factuality, latency, cost), routes each run to the cheapest model that clears the quality bar, competitions the best candidates against each other, and feeds the results back into threshold tuning — closing a loop that makes every subsequent run incrementally better.

the self-improvement loop
bonfyre-run ──outputs──▶ bonfyre-control score ← HE-SLI: relevance · completeness · coherence · factuality · latency · cost bonfyre-control route ──consults──▶ bonfyre-economy route ──▶ bonfyre-tier route bonfyre-compete run ──results──▶ bonfyre-learn feedback bonfyre-learn tune ──updates──▶ bonfyre-control policy ← loop closes here bonfyre-time ──fires trigger──▶ bonfyre-run ──re-processes──▶ bonfyre-entity resolve bonfyre-flow ──allocates──▶ bonfyre-space ──shared state──▶ next stage bonfyre-capability match ──finds──▶ binary + model ──route dispatched

The 10 binaries

Each is a standalone C11 binary: SQLite3 WAL backend, single-file source, zero cloud deps.

Central control plane

The nerve center. Scores every artifact on HE-SLI, enforces routing policies, runs A/B competitions, evicts under-performing models, and produces a signed decision log that every other binary can audit.

Temporal pipeline manager

When a model is updated or an entity is resolved, old artifacts need to be re-processed through the new lens. bonfyre-time schedules retroactive re-runs, diffs outputs, fires event triggers, and maintains a full run history.

bonfyre-compete
Stage/model A/B competition

Registers multiple variants of any pipeline stage, runs them in parallel against real inputs, scores the results on composite HE-SLI metrics, and promotes the winner to production. Every decision is logged and auditable.

  • add-variant

  • history

Cost-aware routing

Maintains per-recipe budget caps, tracks actual spend against them, and recommends the cheapest model that still meets the quality bar. Prevents runaway inference costs before they hit the invoice.

Pipelines as graphs. Define stages, branches (conditional routing), merges (fan-in), and loops as JSON; bonfyre-flow stores, validates, executes, and tracks state for every run. bonfyre-run executes individual stages; bonfyre-flow orchestrates them.

Artifact-level reinforcement

Collects outcome signals — user ratings, downstream errors, compete win/loss — and uses rolling averages to automatically tighten or loosen per-stage quality thresholds. Every tune is logged with the before/after values and the reason.

bonfyre-tier
Latency tier management

Assigns every pipeline stage to a latency tier (instant / fast / batch / deep), validates observed latencies against the SLA, logs violations, and surfaces which stages are routinely blowing their budget. bonfyre-control and bonfyre-economy consult tier routing before every dispatch.

  • list / show
  • set
  • route
  • record
  • violations
  • history

Four latency tiers. Every stage, every recipe.

bonfyre-tier enforces SLA contracts at the stage level. bonfyre-control and bonfyre-economy use tier routing to pick the right execution path before dispatching.

Tier SLA Use cases Typical binaries
instant < 10 ms In-memory lookups, cache reads, metadata ops, entity resolution bonfyre-entity, bonfyre-space, bonfyre-tier, bonfyre-capability
fast < 100 ms Local model inference, index queries, tagging, summarization (small) bonfyre-tag, bonfyre-brief, bonfyre-economy, bonfyre-control

HE-SLI: six-dimensional quality scoring.

bonfyre-control uses six metrics to produce a composite quality score for every artifact. Scores feed into routing decisions, compete rankings, and learn thresholds — closing the loop.

MetricDescriptionRangeUsed by
relevanceDoes the output address the intended input?0.0–1.0control, learn, compete
completenessIs important information missing?0.0–1.0control, learn
coherenceIs the output internally consistent?0.0–1.0control, compete
factualityDoes it contradict verifiable facts?0.0–1.0control, learn
latencyDid the stage meet its tier SLA?0.0–1.0tier, economy
costDid the run stay within budget?0.0–1.0economy, control

composite = mean(relevance, completeness, coherence, factuality)

# score an artifact via bonfyre-control bonfyre-control score "output/run-001/transcript.json" \ --relevance 0.91 --completeness 0.88 \ --coherence 0.94 --factuality 0.87 \ --latency-ms 3200 --cost-usd 0.0006 artifact : output/run-001/transcript.json relevance : 0.910 completeness: 0.880 coherence : 0.940 factuality: 0.870 latency : 0.936 (3200ms vs 5000ms SLA) cost : 0.988 ($0.0006 vs $0.05 budget) composite : 0.900 ← PASS (threshold: 0.750)

SQLite schemas. One file per binary.

Every binary owns its own WAL-mode SQLite database under ~/.local/share/bonfyre/<name>.db. Override with $BONFYRE_<NAME>_DB.

End-to-end walkthrough: a self-improving podcast pipeline.

Start with a raw audio file. End with a continuously improving, cost-capped, SLA-enforced pipeline that gets better every time it runs.

  1. Ingest: bonfyre-intake add episode.mp3 → artifact registered, space allocated via bonfyre-space open podcast-ep1.
  2. Capability lookup: bonfyre-capability match "transcribe spoken audio to text" → finds speech-to-text → binary: bonfyre-transcribe, model: whisper-large, tier: batch.
  3. Budget check: bonfyre-economy route podcast → whisper-large at $0.006/min within $0.50 cap. Approved.
  4. Tier check: bonfyre-tier route podcast transcribe → tier=batch, SLA=10s.
  5. Execute: bonfyre-run --recipe podcast --stage transcribe → output written to bonfyre-space put podcast-ep1 transcript ….
  6. Latency log: bonfyre-tier record podcast transcribe 3200 → SLA met ✓.
  7. Cost log: bonfyre-economy cost record podcast transcribe whisper-large 0.0006.
  8. Score: bonfyre-control score output/transcript.json → composite 0.90 → PASS.
  9. Entity resolution: bonfyre-entity resolve voice "Speaker_1" → linked to entity-002 (Alice Chen). bonfyre-time fires entity_resolved trigger.
  10. Feedback: bonfyre-learn feedback run-001 0.90 good → rolling avg updates.
  11. Threshold tune: bonfyre-learn tune transcribe → avg 0.90 > threshold 0.75+0.10 → raise gate by 0.02 → new threshold: 0.77. Logged.
  12. Next run: bonfyre-control route podcast consults updated thresholds, economy budget, tier SLAs → dispatches with refined gate. Loop closed.
