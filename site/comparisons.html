Transcription

vs Deepgram / OpenAI Whisper API

vs Deepgram / OpenAI Whisper API $0 / minute

BonfyreTranscribe runs HCP-enhanced Whisper locally. Speaker segmentation, hallucination filtering via Hyperscan (>1 GB/s), filler detection, and eGeMAPSv02 quality scoring (88 audio features). No API key. No per-minute charge. No privacy tradeoff.

What you pay now

$0.0043–$0.006 per minute

Deepgram's Nova-2 charges $0.0043/minute. OpenAI Whisper API charges $0.006/minute. A 45-minute podcast episode costs $0.19–$0.27 per run. If you process 100 episodes a month: $19–$27/month — plus no speaker diarisation, no quality scoring, no hallucination filtering without additional tools.

What Bonfyre does

HCP-enhanced Whisper, local, 15× real-time

BonfyreTranscribe runs on your hardware, not Deepgram's servers. On M2 Pro it processes 15× faster than real-time — a 45-minute episode takes ~3 minutes. Speaker segmentation is built-in. BonfyreTranscriptClean removes hallucinations and filler words using Hyperscan at >1 GB/s. BonfyreSpeechLoop provides streaming partial transcripts during live recordings.

Privacy difference

Audio never leaves your machine

Deepgram and OpenAI receive your audio files. For client calls, medical intake, legal depositions, and investigative reporting, that's a meaningful data transfer. BonfyreTranscribe processes entirely on-device with the same or better accuracy on supported models. HIPAA-compliant by construction — nothing to audit externally.

CMS

vs Strapi / Contentful

vs Strapi / Contentful (CMS) 1,742× smaller

BonfyreCMS is 287 KB — compared to 500 MB for a standard Strapi install with its npm tree. Dynamic schemas, token auth, REST API, full-text search, and structured logging. Zero npm packages. Cold start is instant.

Strapi baseline

500 MB of node_modules for basic CMS functionality

A default Strapi v4 install with PostgreSQL pulls in ~500 MB of node_modules, requires a Node.js runtime, and needs a JavaScript ecosystem managing its own ongoing security patches. Cold start on a small server: 4–8 seconds. A single npm audit can surface dozens of transitive vulnerabilities.

BonfyreCMS

287 KB, no runtime dependencies

A single statically-linked C binary. Dynamic schemas defined at startup. Token auth using bf_crypto (Ed25519 + BLAKE2b). REST API with full-text search backed by LMDB. Structured JSON logging via bf_log. Cold start: under 20 ms. The server is smaller than a single compressed JPEG. Zero security patches from transitive dependencies.

Contentful / hosted CMS

~$300–$800/month for team plans

Contentful Team is $300/month. Contentful Scale is $800+/month. For most Bonfyre use cases — structured artifact metadata, dynamic content for published pages — BonfyreCMS needs zero monthly cost. The schema changes are versioned. The data stays local.

Vector search

vs Pinecone / Weaviate Cloud

vs Pinecone / Weaviate Cloud 5 ms local search

BonfyreVec + BonfyreEmbed give you NEON SIMD cosine similarity and BM25 hybrid ranking. Embeddings stored in zero-copy LMDB. BonfyreWeaviateIndex connects to self-hosted Weaviate. $70–$250/month hosted → $0/month local.

Pinecone baseline

$70–$250/month for a serverless vector database

Pinecone Serverless starts at ~$70/month for small workloads and scales to $250+/month for team use. Weaviate Cloud is similarly priced. Both require your embedding data to live on their infrastructure, adding a data residency consideration for private or regulated workloads.

BonfyreVec + BonfyreEmbed

NEON SIMD, sub-5 ms, all local

BonfyreEmbed generates sentence-level vectors from BonfyreBrief's structured output. BonfyreVec stores them in zero-copy LMDB and queries them using NEON SIMD cosine similarity (ARM) or AVX2 (x86-64). Retrieval on 100K vectors: under 5 ms. BonfyreIndex adds BM25 full-text search to the same LMDB backend. BonfyreQuery unifies all three retrieval strategies in one call.

Self-hosted Weaviate option

BonfyreWeaviateIndex for larger scale

If you need Weaviate's GraphQL interface or larger-scale ANN indexing, BonfyreWeaviateIndex connects to a self-hosted Weaviate instance. You get Weaviate's full feature set on your own hardware at $0/month in hosting fees. The embedding generation still happens via BonfyreEmbed — no OpenAI embedding API needed.

Realtime media

vs Twilio / LiveKit

vs Twilio / LiveKit (realtime) $0 / stream

BonfyreMoQ is a pure C WebTransport/MoQ relay for private realtime audio and media fan-out. No SaaS relay bill, no Node.js in the media path, no per-stream markup. Inline AI scoring via BonfyreTone and BonfyreSpeechLoop.

Twilio baseline

$0.004/min per participant for Voice

Twilio Voice charges $0.004/minute per connected participant. A 10-person meeting running 60 minutes costs $2.40. At 100 meetings/month that's $240/month. LiveKit Cloud charges per GB of media relayed. Both require your audio to transit their infrastructure, which rules out most private or regulated use cases.

BonfyreMoQ

QUIC + WebTransport relay, your infrastructure

MoQ Transfork over QUIC. The relay runs on your own hardware. No audio transits a SaaS provider. Inline AI scoring via BonfyreTone (eGeMAPSv02, 88 audio features) and BonfyreSpeechLoop (streaming partial transcripts) adds under 20 ms to latency budget without routing through a cloud inference endpoint. See the full BonfyreMoQ reference for transport detail.

WebRTC alternative

No STUN/TURN overhead

WebRTC-based solutions require STUN/TURN servers for NAT traversal and complexity grows with fan-out size. MoQ over QUIC provides a simpler model — multiplexed streams over a single QUIC connection with surface-first priority. No SFU cluster to manage.

Bonfyre coverage

2.1 MB total. No monthly bills.

BonfyreAuth replaces Auth0. BonfyrePay replaces Stripe's platform layer. BonfyreMeter replaces Segment. BonfyreCMS replaces Contentful. BonfyreVec replaces Pinecone. BonfyreProxy provides an OpenAI-compatible local endpoint. BonfyreMoQ replaces Twilio. BonfyreOffer replaces Gumroad. BonfyreFinance tracks margins. The limit

What Bonfyre doesn't replace

Bonfyre doesn't replace a global CDN for high-traffic assets, a managed database for multi-tenant SaaS at scale, or a team collaboration platform. It replaces the processing, structuring, and serving layer for content and media workflows. If your backend is mainly doing what Bonfyre does — process audio, structure content, serve APIs, meters usage — the substitution is direct.

AI inference

vs cloud GPU / hosted model APIs

vs cloud GPU for AI inference Runs on your laptop

FPQ 4× compression means a model that required a $10K cloud GPU runs on a $500 laptop instead. 99.97% cosine similarity preserved. 15 models already published on HuggingFace. KV cache compression gives 4× more context in the same VRAM.

Cloud AI glue Local proxy + planner

Use an OpenAI-compatible local endpoint and a machine-only planner layer instead of wiring every workflow through paid cloud APIs and brittle app glue.

Cloud GPU cost

$2,000–$8,000/month for sustained inference

A single A100 80GB GPU runs $2.00–$3.50/hour on AWS, GCP, or Azure. Sustained inference for a 70B parameter model at moderate load: 4–6 GPUs, $2,000–$8,000/month. Per-API pricing (GPT-4 Turbo: $0.01/1K input tokens) adds up fast once you're running multi-step pipelines against a 45-minute transcript.

FPQ result

Llama-3-70B on a 6 GB consumer GPU

BonfyreFPQ compresses the model to INT8 with polynomial correction. 4× reduction means Llama-3-70B's ~140 GB full-precision footprint becomes ~35 GB. With KV cache compression: down to 6 GB VRAM — a $350 RTX 3060. On M-series MacBooks the unified memory architecture makes this even more practical. See the full BonfyreFPQ reference for hardware fit details.

BonfyreProxy

OpenAI-compatible local endpoint

BonfyreProxy exposes an OpenAI-compatible API locally. Any code that calls openai.ChatCompletion.create() runs against your local FPQ model without changing a line. BonfyreRuntime's planner layer handles multi-step reasoning chains locally — no LangChain dependency, no cloud routing per planning step.
