New capabilities

The newest systems work lives below the product copy.

These additions are the ones that materially changed Bonfyre’s backend story: a new reactor path for the API, hot-load pipeline stages, QUIC-aware artifact transport, a swarm binary, faster SIMD decode paths, and an HE-compatible SLI bridge.

bf_reactor

Shared-nothing API runtime

Per-core event loop with kqueue on macOS, io_uring or epoll on Linux, SO_REUSEPORT, token-bucket limiting, SSE streaming, and 4,096 connection slots per reactor.

bf_hotload

Plugin-loaded pipeline stages

Hot-load registry with dlopen, dlsym, watcher-based reloads, ABI checks, up to 32 loaded stages, and A/B execution routed by FNV-1a hash of the pipeline id.

bonfyre-distribute

QUIC family streams

The distribution layer now has an in-tree QUIC send path with artifact-family multiplexing, surface-first prioritization, and reconnect behavior designed for sync workloads instead of plain TCP webhook fan-out alone.

bonfyre-swarm

BitTorrent v2 artifact CDN

Swarm delivery uses SHA-256 piece trees, 256 KB pieces, BonfyreIndex as local peer knowledge, and Gate plus Meter as swarm credential and accounting hooks.

simd_bp128

128-int SIMD decode blocks

The FPQ tree now includes SIMD BP128 for E8 INT7 coordinates, with NEON and AVX2 decode paths aimed at making SLI decode latency feel more like memory bandwidth than byte-by-byte entropy decode.

he_sli

Encrypted linear scoring path

The HE-SLI bridge validates the architectural claim that SLI scoring is linear enough to fit an HE-friendly path with multiplicative depth 1, even before a full production HE backend lands.
