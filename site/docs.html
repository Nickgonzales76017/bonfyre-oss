Complete inventory by functional layer.

Each binary handles one job, links only the C it needs, and communicates via content-addressed artifacts.

Layer 1 — Ingest & signal

Getting audio and media into the pipeline

Before any model sees your audio, these binaries normalise it. BonfyreMediaPrep resamples to 16 kHz and loudness-normalises to broadcast standard so Whisper runs at peak accuracy. BonfyreSegment identifies speaker boundaries so the transcript has [Host] and [Guest] attribution rather than a word dump. BonfyreSpeechLoop and BonfyreTone score quality in real time using 88 eGeMAPSv02 features — so you know before the pipeline advances whether the recording is usable. BonfyreHash fingerprints every cleaned file: if you re-run the same pipeline tomorrow with the same recording, every step that hashes to the same value is skipped automatically.

Layer 2 — Transcription & cleanup

Converting audio to clean, structured text

BonfyreTranscribe runs a local HCP-enhanced Whisper. No API key. No per-minute charge. No audio leaving your machine. On an M2 Pro it processes roughly 15× faster than real-time. BonfyreTranscriptClean then passes the raw output through Intel Hyperscan pattern matching at over 1 GB/s to remove hallucinations (Whisper-invented phrases), filler words, false starts, and repeated words. What you get is a clean, legible transcript. BonfyreParagraph groups it into readable sections with implied headings. BonfyreTranscriptFamily clusters multiple related recordings so a series of interviews or episodes can be searched as a single family.

Layer 3 — Knowledge & memory

Structuring, tagging, and making things findable

This is the layer that turns a transcript into usable memory. BonfyreBrief extracts a structured summary: title, topics, key decisions, action items, speaker breakdown. BonfyreEmbed generates sentence-level vectors from that brief. BonfyreVec stores them in a zero-copy LMDB index and uses NEON SIMD cosine similarity for sub-5 ms semantic retrieval. BonfyreQuery exposes a single interface across full-text search, vector similarity, and raw key-value lookups so you don’t have to choose a single retrieval strategy. BonfyreGraph maps relationships between artifacts — not just "find this record" but "find everything related to this record across all recordings." BonfyreKVCache is the zero-copy hot-read path that every downstream step goes through.

Layer 4 — Production & publishing

Turning structured data into surface-ready outputs

Input is a structured brief. Output is something you can hand someone. BonfyreRepurpose derives multiple formats (show notes, social clips, email copy, LinkedIn update) from one brief in a single model pass — not four separate API calls. BonfyreRender builds an HTML page. BonfyreEmit produces PDF, EPUB, JSON, and RSS simultaneously from the same source data. BonfyrePack bundles all outputs into a single content-addressed archive with a BLAKE2b manifest. BonfyreProof signs that bundle with an Ed25519 key so any recipient can verify it came from you and hasn’t been modified since. BonfyreCMS is a 287 KB standalone HTTP server with dynamic schemas, token auth, full-text search, and structured logging — for when you need a structured content API without pulling in npm.

Layer 6 — Platform & distribution

Runtime, networking, auth, and delivery infrastructure

This layer makes the other five composable at scale. BonfyrePipeline registers up to 32 stages and hot-swaps them without restarting the host process — push a new stage binary, atomic-swap the function pointer, and the next artifact routes through the new code. BonfyreQueue buffers work between stages. BonfyreOrchestrate coordinates multi-pipeline flows. BonfyreSwarm delivers artifacts peer-to-peer via BitTorrent v2 SHA-256 piece trees. BonfyreDistribute uses fountain codes over QUIC so up to 33% packet loss is recoverable without a full retransmit. BonfyreProxy exposes an OpenAI-compatible HTTP endpoint on localhost: any existing tool that speaks the OpenAI API routes automatically to your local FPQ-compressed model instead of paying per token.

26-module runtime

The substrate every binary links against.

No Node.js. No Python. No JVM. Every module is pure C — zero-copy, zero-allocation where measurable, no abstraction overhead. These 26 modules are the shared standard library all 53 binaries link against. A security fix in bf_crypto propagates to all 53 binaries at the next build — not 53 separate dependency PRs. The entire runtime is 180 KB. A default Express.js install pulls in roughly 50 MB of node_modules to cover what three of these modules provide.

BonfyreFPQ

Model compression that fits large models onto your hardware.

FPQ = Fountain-coded Polynomial Quantisation. FP32 → INT8 with SIMD decode, 4× size reduction, 99.97% cosine similarity preserved.

Quantisation

FP32 → INT8 with polynomial correction

Polynomial coefficients correct for the non-linear error introduced by INT8 truncation. Cosine similarity on re-expanded vectors lands at 99.97% of full-precision baseline across the 15 published models.

SIMD decode

AVX2 / NEON vectorised re-expansion

INT8 weights re-expanded to FP32 on the fly during inference using SIMD batched multiply-add. Decode overhead ≤ 2% vs loading uncompressed FP32 weights from disk.

Fountain coding

Erasure-resilient chunk distribution

Model checkpoint shards encoded with fountain codes (256 KB pieces). Any k-of-n chunk subset reconstructs the full checkpoint. Enables peer-to-peer model distribution via BonfyreDistribute without a central server.

KV cache compression

4× more context in the same VRAM

Attention KV cache quantised independently of weights. A model that needed 24 GB VRAM for a long context window now fits in 6 GB. Practically: Llama-3-70B on a 6 GB laptop GPU.

Published models

15 models on HuggingFace

Quantised checkpoints published and downloadable. Each model ships with a companion metadata file that BonfyreFPQ uses to verify decode fidelity before first inference.

Hardware fit

GPU is optional

FPQ models run on CPU-only laptops (M-series, x86-64 with AVX2), consumer GPU (6 BB VRAM), and professional GPU (24 GB VRAM). The limit table tells you which model fits which tier.

BonfyreMoQ — realtime media

Pure C WebTransport relay. No Node.js in the media path.

Transport

QUIC + WebTransport

MoQ Transfork over QUIC. Low-latency media fan-out without WebRTC complexity or SFU cloud bills. Relay runs on your own hardware — no Twilio or LiveKit dependency.

Inline AI scoring

BonfyreTone + BonfyreSpeechLoop in the media path

Audio streams scored for quality (eGeMAPSv02, 88 features) and tone in real time, without routing through a cloud inference endpoint. Latency budget: < 20 ms added to relay path.

Delivery

BonfyreDistribute for model and artifact fan-out

FPQ-compressed model shards delivered peer-to-peer using fountain codes. Tolerates up to 33% packet loss before reconstruction fails. Designed for flaky field network conditions.
