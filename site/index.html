26-module shared runtime

What every binary links against.

The 26-module C library replaces the npm/pip dependencies most backends accumulate. Every binary gets it for free. Zero external dependencies beyond libc.

I/O and concurrency

bf_reactor · bf_pool · bf_pipeline · bf_queue

Per-core event loops with kqueue/io_uring, coroutine-based pipeline execution with channels and deadlines, worker pool with work-stealing, and async artifact queue.

Storage and data

bf_lmdb · bf_mmap · bf_hashmap · bf_csv · bf_flatbuf

Zero-copy LMDB reads (pointer casts directly into mmap), custom open-addressing hashmap, CSV parsing, and FlatBuffer manifest handling for repeated-access hot paths.

Encoding and serialisation

bf_json · bf_msgpack · bf_lz4 · bf_utf8 · bf_base64

SIMD JSON parser, MessagePack serialisation, LZ4 streaming compression, Unicode validation and normalisation, and base64 encode/decode.

Security and identity

bf_crypto · bf_bloom · bf_opts · bf_conf

BLAKE2b hashing, XSalsa20-Poly1305 encryption, Ed25519 signing, Bloom filter for dedup, CLI option parsing, and structured config management.

Networking and HTTP

bf_picohttpparser · bf_log · bf_mimalloc

Zero-copy HTTP/1.1 request parsing, structured levelled logging with caller context, and optional mimalloc heap integration for per-reactor allocation.

Why this matters

One shared library. 180 KB.

The entire runtime fits in 180 KB. Every binary links the same modules. That means one security patch propagates to all 53 binaries at build time, not 53 separate dependency upgrades across 53 package managers.

Recent additions

What got added this cycle.

Swarm delivery, QUIC transport, shared-nothing API reactor, hot-load pipeline stages, zero-copy LMDB cache, and SIMD decode upgrades.

BonfyreSwarm

BitTorrent v2 artifact CDN

SHA-256 piece trees, 256 KB pieces, local peer knowledge, and Gate plus Meter hooks baked in. Artifact bundles don't need a CDN vendor. They self-distribute.

BonfyreDistribute

QUIC family streams

Artifact families multiplex over QUIC with surface-first priority, reconnect logic, and flow control tuned for sync-heavy artifact sets. No separate transport vendor needed.

bf_reactor

Shared-nothing API runtime

Per-core event loops, faster HTTP/1.1 request decode via bf_picohttpparser, SSE push support, and per-reactor mimalloc heaps. 4,096 concurrent connections per reactor core.

BonfyrePipeline hot-load

Swap stages without rebuilding

32-stage plugin registry, 16 live A/B pairs, and a coroutine-based execution path with channels and deadlines. Iterate on a stage without touching the rest of the pipeline.

bf_lmdb + bf_flatbuf

Zero-copy read-hot cache

Repeated artifact fetches pointer-cast directly into LMDB mmap pages. FlatBuffer manifests cut repeated JSON parse overhead. 512 MB default map, 4 named sub-databases.

FPQ SIMD + HE-SLI

Faster decode, private scoring

SIMD BP128 decodes 128 E8 INT7 values per block. HE-SLI adds depth-1 homomorphic encrypted linear scoring — inference stays in the encrypted domain for privacy-critical flows.
