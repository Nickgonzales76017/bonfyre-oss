Binary reference · Layer 1

Ingest & signal.

Before any model sees your audio, these 7 binaries normalise it. They exist because garbage in = garbage out — and a model that receives a 44.1 kHz stereo recording with variable loudness produces noisier transcripts than one that receives a clean 16 kHz mono input with consistent levels. This layer ensures every downstream binary receives a consistent, quality-verified artifact.

Why this layer exists

What actually happens to audio before Whisper sees it.

Sample rate

44.1 kHz podcast files are wrong for Whisper

Whisper was trained on 16 kHz audio. Feeding it a 44.1 kHz file doesn't improve quality — it forces an internal resample that introduces rounding artifacts and slows decode. BonfyreMediaPrep resamples every file to 16 kHz before transcription. This alone reduces hallucination frequency in Whisper output.

Loudness normalisation

Quiet recordings cause more hallucinations

Whisper's hallucination rate triples on recordings below -30 LUFS. BonfyreMediaPrep normalises loudness to broadcast standard (EBU R128 / -23 LUFS) before the file ever reaches BonfyreTranscribe. What sounds like a minor pre-processing detail is the biggest single driver of transcript quality.

Speaker separation

A transcript without speaker labels is half the work

BonfyreSegment runs speaker diarisation before transcription — not after. Knowing speaker boundaries allows BonfyreTranscribe to treat each speaker turn independently, improving attribution accuracy. The output carries [Speaker A] and [Speaker B] labels inline, so every downstream binary knows whose words it's reading.

Content addressing

BLAKE2b fingerprinting makes pipelines idempotent

BonfyreHash writes a BLAKE2b-256 digest for every artifact it touches. If you re-run the same pipeline on the same recording tomorrow, every step that hashes to an existing artifact is skipped automatically — no redundant transcription, no redundant embedding, no redundant publishing. Incremental by default.

Binary reference

All 7 Layer 1 binaries.

BonfyreIngest

Multi-path audio intake and format normalisation

Accepts audio from disk, URL, stdin, or POSIX pipe. Detects container format (MP3, MP4, WAV, OGG, FLAC, M4A) using magic bytes, not file extension. Extracts the raw audio stream and hands it to BonfyreMediaPrep. Deduplicates on BLAKE2b hash — if the same audio arrives via two different paths it's processed once.

BonfyreMediaPrep

Pre-processing for transcription and encoding

Resamples to 16 kHz mono. Applies EBU R128 loudness normalisation to −23 LUFS. Trims leading and trailing silence (configurable threshold, default −50 dBFS). Optionally applies a high-pass filter at 80 Hz to remove low-frequency noise common in room recordings. Writes the normalised file and its BLAKE2b hash.

BonfyreSpeechLoop

Real-time speech quality scoring loop

Calculates 88 eGeMAPSv02 acoustic features per 25 ms frame: loudness, spectral flux, shimmer, jitter, formant bandwidth, and others. Produces a per-file quality score between 0–1. Scores below a configurable threshold (default 0.4) flag the recording for review before pipeline continues — preventing a low-quality file from consuming transcription compute. Processes in real time as audio plays or batch after ingest.

BonfyreTone

88-feature audio quality and tone classification

Shares the eGeMAPSv02 feature set with BonfyreSpeechLoop but focuses on tone classification: detects anger, distress, sarcasm, and disengagement markers. Outputs a tone vector alongside the transcript. BonfyreBrief downstream can note "speaker B showed elevated stress markers in segment 3–7." Used in support call pipelines where emotional context matters.

BonfyreSegment

Speaker segmentation and diarisation

Identifies speaker boundaries using embedding distance on overlapping 1.5-second windows. Produces a speaker-labelled segment map: timestamp ranges attributed to Speaker A, Speaker B, etc. This map is consumed by BonfyreTranscribe for per-speaker transcription and by BonfyreBrief for speaker-aware summary extraction. Handles up to 8 simultaneous speakers.

BonfyreHash

BLAKE2b-256 content addressing for all artifacts

Computes BLAKE2b-256 digests for every artifact produced by the pipeline. Writes a .hash manifest alongside each artifact. Downstream binaries check the manifest before processing — if the artifact already exists with a matching hash, the step is skipped. This is what makes a 15-recording batch pipeline skip 14 unchanged recordings when one new file is added.

BonfyreCompress

LZ4 + zstd artifact compression

Applies LZ4 for streaming compression (real-time artifact writing) and zstd level 3 for batch archive compression. LZ4 decodes at 4 GB/s — compressed artifacts read by downstream binaries at near-memory speed. Used automatically when BonfyrePack assembles artifact bundles. Can be called standalone to compress any artifact for long-term storage.

What comes next

Layer 1 produces the inputs that Layer 2 consumes.

Output of Layer 1

Clean, normalised, fingerprinted audio

A 16 kHz mono .wav file at −23 LUFS, speaker-segmented, quality-scored, with a BLAKE2b hash. This is what BonfyreTranscribe expects on its input.

Input to Layer 2

BonfyreTranscribe receives a guaranteed-quality file

Because Layer 1 normalises before transcription, transcript accuracy is consistent across different recording environments, microphones, and loudness levels. The model never receives a file that hasn't been quality-checked.

Skip behaviour

Re-runs skip processed files automatically

BonfyreHash means that running the full pipeline on 50 recordings the second time, when only 5 are new, completes the 45 existing hashes in milliseconds and only processes the 5 new files. No redundant work.
