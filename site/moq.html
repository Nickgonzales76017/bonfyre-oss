What it replaces

vs Twilio · LiveKit · Agora.

vs Twilio / LiveKit (realtime) $0 / stream

BonfyreMoQ is a pure C WebTransport/MoQ relay for private realtime audio and media fan-out. No SaaS relay bill. No Node.js in the media path. No per-stream markup. Inline AI scoring via BonfyreTone and BonfyreSpeechLoop runs locally without routing through a cloud inference endpoint.

vs WebRTC SFU servers No STUN/TURN overhead

WebRTC SFU architectures require STUN/TURN servers for NAT traversal and scale complexity grows with fan-out. MoQ Transfork over QUIC provides a simpler transport model — multiplexed streams over a single QUIC connection, with surface-first priority for responsive delivery to heterogeneous clients.

vs cloud AI scoring < 20 ms added latency

Services like AWS Transcribe or Deepgram add 300–800 ms of round-trip latency for quality scoring because audio has to leave the device and come back. BonfyreTone and BonfyreSpeechLoop run inline in the relay path — inside the relay process — adding under 20 ms to the latency budget.

Transport

QUIC + WebTransport + MoQ Transfork.

QUIC transport

Multiplexed streams over a single connection

QUIC multiplexes independent streams over a single UDP connection. No head-of-line blocking — a dropped packet in one audio stream doesn't stall all other streams. QUIC's 0-RTT handshake reconnects sessions in under 1 ms after a brief network interruption.

WebTransport

Browser-compatible without a plugin

WebTransport uses QUIC under HTTP/3 and is natively available in modern browsers. Clients can consume BonfyreMoQ streams directly in the browser — no WebRTC plugin, no media server SDK, no npm package required on the client side.

MoQ Transfork

Surface-first delivery priority

The MoQ Transfork protocol layer manages stream priority. Artifact families multiplex with surface-first ordering — the content being actively displayed gets packets first. Reconnect logic and flow control are tuned for sync-heavy artifact sets where partial data arrival is worse than a brief stall.

Fountain codes for resilience

Tolerates 33% packet loss

FPQ-compressed model shards and artifact chunks are fountain-coded before transmission. Any k-of-n subset of 256 KB pieces reconstructs the full payload. Designed for field network conditions — unreliable mobile links, congested WiFi, overseas routes with high jitter.

Inline AI scoring

Quality and tone scoring inside the relay process.

BonfyreTone

eGeMAPSv02 — 88 audio quality features

eGeMAPSv02 is a standardised vocal feature set: fundamental frequency, jitter, shimmer, formant frequencies, spectral flux, and 82 others. BonfyreTone extracts all 88 in real time. Used for live call quality assessment, speaker mood classification, and post-session analysis without capturing raw audio off-device.

BonfyreSpeechLoop

HCP-enhanced streaming recogniser

Streaming speech recognition running in the relay path. Partial transcripts available within the latency budget of a single QUIC packet round-trip. Combined with BonfyreTone, it gives you quality-scored, partially-transcribed audio in one pass without two separate inference calls.

Privacy model

No audio leaves your infrastructure

AI scoring runs inside the relay process on your own hardware. No audio samples transit a cloud API. No transcript fragments are sent to an external endpoint. You get quality metrics and partial transcripts without a data privacy agreement with a third-party AI vendor.

What plugs in

MoQ is one binary in a larger pipeline.

Gate + Auth

Stream-level access control

BonfyreGate sits in front of the relay. Token-authenticated connections. Ed25519-signed session tokens via bf_crypto. No dependency on a cloud auth provider — the same Auth + Gate pair used by BonfyreCMS and BonfyreAPI.

Meter

Per-stream usage events

BonfyreMeter records stream events to LMDB. Minute-granularity usage data for billing, analytics, or rate limiting. BonfyreLedger picks up those events for the immutable financial log. No third-party analytics SDK in the realtime path.

BonfyreDistribute

Model and artifact fan-out

FPQ-compressed model shards delivered peer-to-peer using fountain codes. The same transport layer that carries media also carries model weights and artifact bundles — no second CDN needed for AI assets alongside live media.

Post-session pipeline

Live session → packaged deliverable

When a session ends, the relay hands off to the standard pipeline: BonfyreTranscribe → BonfyreBrief → BonfyreRepurpose → BonfyreEmit → BonfyrePack → BonfyreProof. The same pipeline that processes pre-recorded audio handles live sessions without a separate code path.
