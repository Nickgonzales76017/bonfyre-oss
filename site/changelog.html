Current cycle — Cycle 3

Transport, runtime, compression, and private inference.

Six additions this cycle: peer-to-peer artifact distribution, QUIC transport, a shared-nothing API reactor, hot-load pipeline stages, a zero-copy LMDB cache layer, and SIMD decode plus homomorphic scoring for the FPQ model compression layer.

New binary — BonfyreSwarm

BitTorrent v2 artifact CDN

SHA-256 piece trees, 256 KB pieces, and local peer knowledge maintained in LMDB. Gate and Meter hooks baked in — access to swarm content can be quota-guarded without a separate policy service.

Why it matters: Artifact bundles don't need a CDN vendor. They self-distribute to peers who already have pieces. Large model bundles and archive packs distribute to a team without a CDN subscription or S3 bucket.

New binary — BonfyreDistribute

QUIC family streams with fountain codes

Artifact families multiplex over QUIC with surface-first priority and reconnect logic. LT-code fountain coding means up to 33% packet loss is recoverable without a full retransmit.

Why it matters: Field researchers syncing archives over satellite links no longer get incomplete transfers on lossy connections. BonfyreDistribute + fountain codes make lossy sync reliable.

Runtime addition — bf_reactor

Shared-nothing API runtime

Per-core event loops with faster HTTP/1.1 decode via bf_picohttpparser, SSE push support, and per-reactor mimalloc heaps. 4,096 concurrent connections per reactor core.

Why it matters: Every binary that exposes an HTTP surface — BonfyreAPI, BonfyreProxy, BonfyreCMS — runs on bf_reactor. Per-core shared-nothing means no lock contention across cores. Throughput scales linearly with core count.

Pipeline addition — BonfyrePipeline hot-load

Swap stages without rebuilding

32-stage plugin registry, 16 live A/B pairs, and a coroutine-based execution path with channels and deadlines. Drop a new stage binary into the directory, call the hot-swap endpoint. The swap is atomic at the function pointer level.

Why it matters: Iterating on a transcription cleanup pass or a brief extraction prompt no longer requires restarting the pipeline. Swap the stage in; the next artifact routes through the new code. A/B pairs make staged rollouts safe.

Runtime addition — bf_lmdb + bf_flatbuf

Zero-copy read-hot cache

Repeated artifact fetches pointer-cast directly into LMDB mmap pages. bf_flatbuf manifest fields are accessed via direct struct overlay — no JSON parse overhead on repeated reads. 512 MB default map, 4 named sub-databases.

Why it matters: Pipeline stages that read the same brief multiple times (BonfyreRepurpose generating 4 formats) do zero heap allocation per read after the first. At high batch volumes, this is the difference between memory pressure and not.

FPQ addition — SIMD BP128 + HE-SLI

Faster decode and private scoring

SIMD BP128 decodes 128 E8 INT7 values per instruction block. HE-SLI adds depth-1 homomorphic encrypted linear scoring — inference stays in the encrypted domain for privacy-critical flows.

Why it matters: SIMD BP128 accelerates FPQ decode on every inference call. HE-SLI enables scoring pipelines where the input data is sensitive — the model scores over encrypted representations, not cleartext. Legal and health data pipelines specifically.

Cycle 1

Core pipeline: ingest through publishing.

The first cycle built the core data pipeline: audio in → transcript clean → brief extracted → page published. Layers 1 through 4, the 26-module shared runtime, and the FPQ model compression baseline.

Layers 1–4

Ingest → Transcribe → Brief → Publish

BonfyreIngest + MediaPrep + Segment → BonfyreTranscribe + TranscriptClean + Paragraph → BonfyreBrief + Tag → BonfyreRender + Emit + Pack + Proof. The core pipeline from untouched audio to a signed, published artifact bundle.

26-module runtime

The shared substrate

bf_reactor, bf_lmdb, bf_mmap, bf_hashmap, bf_json, bf_msgpack, bf_lz4, bf_crypto, bf_bloom, and 17 more. 180 KB total. Every binary links the same library. A security fix propagates to all 53 binaries at the next build.

FPQ baseline

INT8 quantisation with polynomial correction

4× compression of FP32 model tensors to INT8 with polynomial correction coefficients. 99.97% cosine similarity to the original. First 4 models published on HuggingFace. GPU optional — CPU inference via SIMD decode on Apple Silicon and x86 AVX2.
