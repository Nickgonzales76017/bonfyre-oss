## Decisions
- templates should force clearer buyer, proof, and metrics fields
- monetization notes deserve their own folder and schema
- capture major implementation changes before moving on when the delta is meaningful
- keep implementation moving between captures unless a major shift or decision point appears
- when a new capability overlaps an existing system, merge into the shared layer first and document reuse before building sideways
- if a local job is heavy, it should honor the shared runtime lock and load guard before it runs
- local subprocess wrappers should own timeout and cleanup, especially around llama.cpp
- heavy generated/model directories should be excluded from indexing by default
- keep the handoff contract agnostic, then add smoother packaging on top of it
- if the heavy step is already done, use rebuild-style iteration before spending more CPU on another full transcription pass
- if a new intake package arrives during a busy session, enqueue it instead of forcing immediate execution
- if a transcript is long, summarize it through chunked section aggregation first and let the flat executive summary inherit from that stronger structure
- if a tool already writes a generated operating note, downstream analytics should read that note or its paired snapshot first
