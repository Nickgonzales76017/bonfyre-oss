# Concept: Automation

## Summary
Automation is the transfer of repeated work from memory and manual effort into a reliable process.

## Definition
Automation in this vault includes templates, scripts, shortcuts, routing logic, and AI-assisted transforms. It does not need to be fully hands-off to be useful.

## Why It Matters
- it protects attention for higher-value work
- it reduces inconsistency
- it makes small systems feel much larger

## Signals
### Positive Signals
- recurring work gets faster
- fewer steps depend on memory
- the same input tends to produce the same output

### Failure Signals
- automation adds more setup than it saves
- edge cases constantly break the flow
- it is built before manual truth exists

## Where It Applies
- projects: transcription formatting, outreach generation, logging
- systems: every repeated delivery or review step
- pipelines: note routing, external tools, AI plus human review
- daily execution: reducing repeated decision fatigue

## Decisions This Concept Improves
- what to templatize
- what to automate only after manual proof
- which steps should stay manual for quality

## Examples
### In This Vault
- folder templates in Obsidian
- daily note generation
- standardized delivery notes

### In The Real World
- email sequences
- file watchers
- workflow tools like n8n

## Tradeoffs
- upside: speed and consistency
- downside: false confidence when a bad process is automated
- common misuse: automating before understanding the real work
