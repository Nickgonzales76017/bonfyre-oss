# Idea: Batch Job Runner

## Summary
Add batch execution so multiple files can be processed consistently without manual command repetition.

## Core Insight
Single-file flows are useful for proof, but batch throughput is where a real service starts feeling operational.

## First Use Case
Drop three transcript or audio inputs into a folder and generate three structured deliverables in one run.

## Why It Might Work
- inefficiency: manual per-file runs create friction fast
- arbitrage: batching increases throughput with little extra complexity
- trend: operator-focused tooling values queue-like workflows
